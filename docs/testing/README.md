# Testing Documentation

This folder contains testing guides, reports, and documentation for the Nexus project.

## Structure

```
docs/testing/
├── README.md                 # This file
├── enforcer/                 # Enforcer-specific tests
│   └── testing-guide.md
├── workspace/                # Workspace-specific tests
│   └── testing-guide.md
├── plugins/                  # Plugin testing
│   └── plugin-testing.md
└── reports/                  # Testing reports
    └── dogfooding-reports/
```

## Document Types

- **Testing Guides** - How to test specific components
- **Dogfooding Reports** - Real-world usage reports
- **Manual Test Plans** - Step-by-step manual testing procedures

## Guidelines

1. Keep test documentation focused on current implementation
2. Archive outdated test reports after 6 months
3. Link test plans to actual test files when possible
