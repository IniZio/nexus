The parity matrix lists test IDs for core product-critical flows that both the UI and CLI are expected to support end-to-end: runtime selection, worktree sync, lifecycle hooks, spotlight compose, and tools auth forwarding. It is enforced by `cases/ui-cli-parity-map.test.ts`, which requires every matrix ID to appear in the flows case implementations. Cases that exercise live credentials, extended CLI lifecycle, or other specialized environments are intentionally not listed here—examples include `tools-auth-live` and `cli-runtime`, which depend on optional env or full CLI builds rather than the shared parity contract.
